读书笔记之Array方法总结

最近开始看书复习 js 基础,看到 Array 一章，Array 是 javascript 最常见的数据结构，也是使用仅次于 Object 的引用类型，但平时使用的时候却很容易记忆混淆。正好最近开始搭建博客，就用这一篇基础的知识作为博客练手文章，也方便自己和大家能更好的记忆区分 Array 各种方法的使用区分。在这部片文章里，我将根据这些方法的操作作用来分类讲解，并辅以代码讲解方法的使用区分。

栈方法

javascript 数组提供了一系列让数组行为类似其他数据结构的方法。
比如栈是一种可以限制插入和删除项的数据结构，是一种 LIFO (Last-In-First-Out,后进先出)的结构，也就是最新添加的项最后被移除。栈中项的插入和已出都只发生在栈的顶部。js 提供了push() 和 pop() 方法，可以实现类似栈的行为。

  • push(): 可以将接收任意数量的参数，并将它们逐个添加到数组末尾，返回修改后数组的长度

  • pop(): 从数组末尾移除最后一项，减少数组的length值，然后返回移除的项。

队列方法

与栈相对应的数据结构是队列，队列的访问规则是 FIFO(Firt-In-First-Out,先进先出),前面的 push() 方法仍然可以用于向队列末尾项推入元素，而 javascript 数组还提供了一个方法 shift() 用于移除数组的第一项。

  • shift(): 移除数组的第一项元素，数组长度减1，并返回该项

    以上就实现了队列数据结构的访问规则。与 shift() 相对应的，javascript 数组还提供了 unshift() 方法，顾名思义与 shift() 相反

  • unshift(): 接收任意数量的参数，向数组的前端添加新的元素，并返回数组新长度

重排序方法

javascript 数组存在两个可以直接排序的方法。

  • reverse()：将数组反转，返回反转顺序后的数组。

  • sort():按照传入的顺序函数的规则将数组进行排序。返回进过排序后的数组。

    从上面例子可以看出 sort() 方法并非按照数字值进行排序，而是调用每个数组项的 toString() 转型方法，然后比较得到的字符串进行排序。默认情况下，sort() 会按照字符串的升序进行排序。
    这种排序无疑是不理想的，因此 sort() 方法可以接收一个比较函数。
    比较函数接收两个参数，如果第一个参数应该位于第二个参数之前，则返回一个负数；如果两个相等则返回0;如果第一个参数应该位于第二个参数之后则返回一个正数(非数据类型的比较函数？)

操作方法

  • concat():基于当前数组创建一个副本，然后将接受的参数添加到这个副本的末尾，返回新构建的数组
    concat() 没有传递参数的情况下，会复制(深拷贝？)当前数组并返回副本。如果传递的参数是一个或者多个数组，该方法会将这些数组的每一项都依次添加到结果数组中。如果传递的参数不是数组，这些值会被简单的添加结果数组的末尾。(添加引用类型数据？)concat() 方法不改变原数组
  • splice():基于当前数组的一个子片段创建一个新的数组，接收一到两个参数，即要返回项的起始位置和结束位置。返回截取起始位置到结束位置(不包括结束位置)的新数组。
    当slice()方法只接收一个参数时，返回从该参数指定位置项到当前数组末尾的所有项。如果接收两个参数，则返回起始位置和结束位置之间的项，但不包括结束位置的项。slice() 方法不会改变原数组。 (如果传递的值为负数？)
